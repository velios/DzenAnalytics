# Roadmap

Живой список того, что хотим доделать. Не строгий план — приоритеты двигаем по
ситуации. Статусы выверены по коду (а не по комментариям), дата ревизии: 2026-06-12.

## 1.0 — готово к выпуску

Версия в коде поднята до **1.0.0**: весь объём «До 1.0» закрыт и обкатан.
Двусторонняя синхронизация — полностью; все пользовательские пожелания (бюджет
план/факт, FIRE с капиталом, регулярные, хэштеги, счета вне баланса) — внедрены.
Публикация релиза — когда решим выпускать.

### Двусторонняя синхронизация (push) — ГОТОВО ✅
Закрыт весь объём: дата/получатель/бренд/комментарий/сумма/валюта/категория,
смена типа (Расход↔Доход↔Возврат и на/с «Перевод»), смена счёта (в т.ч. счетов
перевода), мультивалютные операции (смена типа на FX-строках, перевод между
счетами разной валюты, перенос на счёт другой валюты, флип FX-операции в перевод —
op-суммы отбрасываются, ноги собираются в валютах счетов), плюс детект конфликтов
перед push. Осталась лишь ниша: редактирование уже существующего FX-перевода с
op-суммами — скипается с понятным сообщением (экзотика).

## Сделано (свежее)

- [x] **Cash-flow и Тренды: свой период** (дефолт «12 мес») вместо глобального
      «месяца» — историю видно сразу при заходе; прочие фильтры работают. Новый
      компонент `PeriodPills` (1.0).
- [x] **Версия в футере + история изменений в приложении**: модалка «История
      изменений» рендерит весь `CHANGELOG.md` (все версии) + ссылка на GitHub;
      версия из `package.json` через `define` (1.0).
- [x] Справка: «Тренды»/«Цели» переведены в правильную группу меню («Ещё»).

- [x] **Бюджет: план/факт по месяцам, периодичность, даты, доход+расход** (запрос
      Elena L). Модель `Record<cat,number>` → `BudgetLine[]` (kind, recurrence
      monthly/quarterly/yearly/once, окно startMonth..endMonth, per-month overrides)
      с авто-миграцией. Переключатель месяца, две секции (расходы/доходы с зеркальной
      семантикой), инлайн-правка плана месяца + «сделать нормой», квартальные/годовые
      падают на один месяц. Чистые хелперы `src/lib/budgets.ts` + 13 тестов; бэкап
      (`budgetsV2`).
- [x] **FIRE: учёт капитала + накопительных счетов** (запрос Elena L). «Лет до FIRE»
      считаются от текущего капитала (сумма выбранных счетов, по умолчанию все
      активные, включая `inBalance:false`), а не «с нуля»; селектор счетов с
      персистом, прогресс к цели, «достигнуто» при капитале ≥ цели; пояснение нормы
      сбережений.
- [x] Регулярные платежи: «протухшесть» с учётом периодичности (молчит дольше ~2
      циклов) вместо плоского «старше года»; протухшие не проецируют «Следующий» в
      будущее, помечены бейджем. Фильтр «Только активные» по умолчанию (запрос Elena L).
- [x] Хэштеги: под каждым тегом раскрывается разбивка расходов Категория →
      Подкатегория с суммами (запрос Elena L). +тесты.
- [x] Push мультивалютных (FX) операций: смена типа на FX-строке переносит
      op-amounts между ногами; перевод между счетами разной валюты (второе поле
      «Сумма зачисления» в редакторе, push строит обе ноги в валютах счетов);
      перенос операции на счёт другой валюты (op-сумма берётся из оригинала).
      +новые тесты; перенос уже-FX операции на третий счёт — скип.
- [x] Push смены типа на/с «Перевод» (одновалютные) + смены счетов существующего
      перевода: flip→transfer строит обе ноги, transfer→single схлопывает на один
      счёт; мультивалютные переводы скипаются с понятным сообщением. +12 тестов.
- [x] Push смены счёта операции (в пределах одной валюты) — оба плеча операции
      переезжают на новый счёт; кросс-валютный перенос пока скипается с понятным
      сообщением.
- [x] Детект конфликтов перед push: тянем свежий diff, и если операцию изменили
      в облаке после нашей синхронизации — правка не отправляется (остаётся в
      очереди), чтобы не затереть чужое изменение.
- [x] Юнит-тесты на фильтрацию: `applyFilters` (период/пресеты/категория/счёт/
      валюта/поиск/пересечения/excludeTransfers) + `presetToRange` + редьюсеры
      стора — 27 тестов.
- [x] Аудит тултипов в тёмной теме: проверены все 16 графиков; одно-серийные
      бары с раскраской через `<Cell>` (Cashflow «сезонность», Trends «по дням
      недели») получили читаемый текст подсказки — как раньше «Категории».
- [x] Доки про двусторонний синк приведены в актуальное состояние: README и
      диалог подтверждения в ImportPage больше не утверждают, что смена типа не
      передаётся; требования к Node в README обновлены под Vite 8.
- [x] Календарь: тепловая карта листается по всем годам, не схлопывается
      глобальным фильтром «текущий месяц» (v0.9.8).
- [x] Экран «Нет данных»: выбор источника — API-синхронизация или CSV (v0.9.8).
- [x] Self-hosting через Docker + онлайн-версия (v0.9.8).
- [x] Тёмная тема: читаемый тултип на графике «Категории» (v0.9.8).
