# Roadmap

Живой список того, что хотим доделать. Не строгий план — приоритеты двигаем по
ситуации. Статусы выверены по коду (а не по комментариям), дата ревизии: 2026-06-09.

## До 1.0

### Двусторонняя синхронизация (push) — закрыть оставшиеся пробелы
Что уже отправляется в облако: дата, получатель, бренд, комментарий, сумма,
валюта, категория/подкатегория, **смена типа** Расход ↔ Доход ↔ Возврат и
**на/с «Перевод»**, **смена счёта** (в т.ч. счетов перевода) в пределах одной
валюты. Остаётся (всё — про другую валюту):

- [ ] **Мультивалютные операции** при смене типа — пропускаются (операционные
      суммы в валюте, отличной от счёта).
- [ ] **Мультивалютные переводы** (между счетами разной валюты) — пропускаются.
- [ ] **Перенос на счёт в другой валюте** — пропускается (нужны op-amounts/FX).

## После 1.0 / идеи

- [ ] **Cashflow / Trends — мягкий дефолт периода.** Это графики «по месяцам/по
      истории», но при заходе показывают текущий месяц (глобальный фильтр). Не
      баг, но можно дать им свой период-селектор (как в Календаре), чтобы на
      старте показывалась осмысленная история.

## Сделано (свежее)

- [x] Push смены типа на/с «Перевод» (одновалютные) + смены счетов существующего
      перевода: flip→transfer строит обе ноги, transfer→single схлопывает на один
      счёт; мультивалютные переводы скипаются с понятным сообщением. +12 тестов.
- [x] Push смены счёта операции (в пределах одной валюты) — оба плеча операции
      переезжают на новый счёт; кросс-валютный перенос пока скипается с понятным
      сообщением.
- [x] Детект конфликтов перед push: тянем свежий diff, и если операцию изменили
      в облаке после нашей синхронизации — правка не отправляется (остаётся в
      очереди), чтобы не затереть чужое изменение.
- [x] Юнит-тесты на фильтрацию: `applyFilters` (период/пресеты/категория/счёт/
      валюта/поиск/пересечения/excludeTransfers) + `presetToRange` + редьюсеры
      стора — 27 тестов.
- [x] Аудит тултипов в тёмной теме: проверены все 16 графиков; одно-серийные
      бары с раскраской через `<Cell>` (Cashflow «сезонность», Trends «по дням
      недели») получили читаемый текст подсказки — как раньше «Категории».
- [x] Доки про двусторонний синк приведены в актуальное состояние: README и
      диалог подтверждения в ImportPage больше не утверждают, что смена типа не
      передаётся; требования к Node в README обновлены под Vite 8.
- [x] Календарь: тепловая карта листается по всем годам, не схлопывается
      глобальным фильтром «текущий месяц» (v0.9.8).
- [x] Экран «Нет данных»: выбор источника — API-синхронизация или CSV (v0.9.8).
- [x] Self-hosting через Docker + онлайн-версия (v0.9.8).
- [x] Тёмная тема: читаемый тултип на графике «Категории» (v0.9.8).
