# Changelog

Все заметные изменения проекта. Формат — по мотивам [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/);
версии — по [SemVer](https://semver.org/lang/ru/).

## [0.9.9] — 2026-06-12

### Бюджет — полностью переписан
- Помесячный **план/факт** с переключателем месяца (свой календарь выбора месяца,
  без нативного браузерного).
- **Периодичность**: ежемесячно / ежеквартально / ежегодно / разово; квартальные и
  годовые суммы «падают» на один месяц периода, а не размазываются.
- **Окно действия** — даты «с» и «по» (строка показывается только внутри окна).
- **Расходные и доходные** бюджеты — две секции с зеркальной семантикой.
- Точечная правка плана на конкретный месяц + «сделать нормой»; кнопка
  **«Копировать с прошлого месяца»**.
- Виджеты с явными **«Факт» и «План»** (расходы / доходы / дельта); полоски прогресса
  по виду строки — доходы зелёные, расходы красные.
- Авто-миграция со старого формата `категория → лимит`.

### Цели / FIRE
- «Лет до FIRE» считаются **от текущего капитала** (сумма выбранных счетов, включая
  накопительные и помеченные «вне баланса»), а не «с нуля»; селектор счетов;
  «достигнуто» при капитале ≥ цели — это убрало «бесконечность».
- Накопительные счета различаются по реальному флагу `savings` из API.

### Счета вне баланса
- Глобальная настройка (**Настройки → Обработка**) — показывать и **учитывать** счета
  `inBalance:false` в списках (Дашборд, «Счета») и в «Совокупном балансе».

### Регулярные платежи
- «Протухшие» определяются **по периодичности** (молчат дольше ~2 циклов), а не плоско
  «старше года»; такие больше не проецируют «Следующий» в будущее.

### Редактор операций и синхронизация
- Переводы — **без категории/подкатегории**; поля «Отправлено» / «Получено» с
  **авто-пересчётом по курсу** синхронизации (редактируемо).
- При смене типа на «Перевод» категория автоматически становится **«Перевод»**.
- Push: **флип мультивалютной операции в перевод** теперь уходит в облако.
- Кнопка **«Убрать зависшие правки»** — чистит осиротевшие правки (остаются после
  перехода с CSV на API, когда меняются id).

### Прочее
- Правильное склонение чисел в интерфейсе (1 правка / 2 правки / 5 правок).
